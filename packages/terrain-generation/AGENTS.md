# terrain-generation Agent Notes

## Purpose

This package is the Rust/WASM terrain height-field engine used by the game to build planetary mesh chunks. Treat it as
a performance-critical runtime package with a public JavaScript API generated by `wasm-bindgen`.

The game calls this package from the terrain worker pipeline under
`packages/game/src/ts/frontend/universe/planets/telluricPlanet/terrain`. Any change here can affect chunk generation,
LOD seams, vegetation/scatter placement, normals, and Playwright visual snapshots.

## Public API Contract

Be careful with exported `#[wasm_bindgen]` types and functions:

- `build_chunk_vertex_data`
- `BuildData`
- `ReturnData`
- `TerrainSettings`
- `Direction`

Changing constructor argument order, field names, numeric types, buffer semantics, or exported names is a breaking
change for the game-side TypeScript worker code and for the published npm package. If an API change is unavoidable,
update the generated `pkg/` artifacts, TypeScript call sites, README examples, and tests in the same change.

## Buffer Ownership

The JavaScript caller owns the buffers; Rust fills them in place. Do not allocate replacement output buffers inside the
hot path unless there is a clear performance reason and the JS contract is updated.

`build_chunk_vertex_data` supports two valid mesh buffer layouts:

- base chunk only: `resolution * resolution` vertices and `(resolution - 1)^2 * 2 * 3` indices,
- base chunk plus skirt: base buffers plus `4 * resolution` skirt vertices and `4 * (resolution - 1) * 2 * 3` skirt indices.

Keep these size formulas in sync with game-side chunk allocation and tests. Invalid buffer sizes should fail loudly
rather than silently writing partial geometry.

## Determinism

Terrain output should be deterministic for the same `BuildData`, `TerrainSettings`, seed, resolution, direction, and
buffer layout. This matters for save stability, visual tests, and reproducible planet surfaces.

Be cautious with global or thread-local state. The terrain function cache in `src/lib.rs` is keyed by seed only; if
terrain settings start varying for the same seed, the cache key must include every setting that affects terrain output.

Avoid nondeterministic scatter behavior unless the caller-facing contract explicitly accepts it. If changing scatter,
add tests around instance counts, buffer bounds, and repeatability expectations.

## Geometry Invariants

Preserve these invariants when touching mesh generation:

- normals must be finite and normalized enough for Babylon lighting,
- indices must stay within the vertex buffer,
- triangle winding must remain compatible with the game material/culling expectations,
- skirt triangles must hide LOD cracks without inverting edge winding,
- average height must remain finite,
- instance matrix writes must never exceed provided buffers.

The terrain code intentionally recenters vertices around the chunk sphere position to reduce floating-point precision
issues. Do not remove that offset without checking the game-side floating origin and high-altitude rendering behavior.

## Tests

Run the package checks when Rust tooling is available:

```sh
pnpm --filter terrain-generation format
pnpm --filter terrain-generation lint
pnpm --filter terrain-generation test:unit
```

For public API or WASM changes, also run:

```sh
pnpm --filter terrain-generation build
```

Then inspect generated `pkg/` diffs deliberately. The `.wasm` artifact is Git LFS-tracked and should only change when
the Rust/WASM output intentionally changes.

Add or update Rust tests under `tests/` for geometry, buffer sizes, deterministic output, terrain layers, noise, and
skirt behavior. Manual visualization tests should remain ignored unless the user explicitly asks to run them.

## Documentation

Keep `README.md`, `package.json`, `Cargo.toml`, and generated `pkg/package.json` aligned when versions, exports,
build workflow, or consumption mode change.

The root project currently uses this package as a workspace dependency. Do not describe a different consumption mode
unless the manifests are changed at the same time.
